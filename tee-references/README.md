## Key Concepts

### Account Encumbrance
- **TEE_HEE:** Proving exclusive ownership of accounts via TEE
- **hell.tech:** Deal-scoped encumbrance with social contracts

### DevProof
- Security: "Can attacker break in?"
- DevProof: "Can developer rug users?"
- Solution: On-chain authorization + timelocks

### ERC-733 Stages
- Stage 0: Ruggable (operator can push any code)
- Stage 1: DevProof (upgrade transparency + exit mechanisms)
- Stage 2: Decentralized (multi-party control)
- Stage 3: Trustless (multi-vendor verification)
